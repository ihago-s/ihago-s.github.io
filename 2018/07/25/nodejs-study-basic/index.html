Node.js学习（1）-基础知识

发表于 2018-07-25 | 分类于 Notes | | 阅读次数:

Node.js是什么？官网给出的解释是：
Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。Node.js 使用了一个事件驱动、非阻塞式 I/O 的模型，使其轻量又高效。Node.js 的包管理器 npm，是全球最大的开源库生态系统。

Node.js中的模块

  • assert- 断言，assert模块提供了断言测试的函数，用于测试不变式
  • Buffer- 缓存，Buffer类在Node.js中是一个全局变量，用于实现二进制数据的存储和转换
  • child_process- 子进程，提供衍生子进程的功能
  • cluster- 集群，用于实现多进程
  • console- 控制台，简单的调试控制台，类似于Web浏览器提供的JavaScript控制台
  • crypto- 加密，提供加密功能，包含对 OpenSSL 的哈希、HMAC、加密、解密、签名、以及验证功能的一整套封装
  • dgram- 数据报，提供UDP数据包scoket的实现
  • dns- 域名服务器，用于实现与dns相关的处理
  • Error- 错误，提供Node.js引起的javaScript错误和系统错误
  • events- 事件，用于为事件处理提供一个基础类
  • fs- 文件系统，用于操作文件和文件系统
  • http- HTTP，用于实现HTTP服务器端及客户端
  • https- HTTPS，用于实现HTTPS服务器端及客户端
  • net- 网络，用于创建TCP服务器与客户端
  • os- 操作系统，提供了一些操作系统相关的实用方法
  • path- 路径，提供了一些用于处理文件和目录路径的工具函数
  • querystring- 查询字符串，用于处理HTTP请求中使用的查询字符串
  • readline- 逐行读取，用于读取一行标准输入流
  • repl- 交互式解释器，用于实现REPL（Read-Eval-Print-Loop）交互式运行环境
  • stream- 流，用于构建实现了流接口的对象
  • string_decoder- 字符串解码器，用于从二进制数据到字符串数据之间的转换
  • tls- 安全传输层，用于使用OpenSSL来实现TLS/SSL通信处理
  • tty- 终端，用于获取来自于TTY终端的信息
  • url- URL，用于URL处理与解析
  • util- 实用工具，主要用于支持Node.js内部API的需求。也可用于应用程序与模块的开发者
  • V8- V8引擎，v8 模块暴露了特定于V8版本内置到 Node.js 二进制文件中的API
  • vm- 虚拟机，vm 模块提供了一系列 API 用于在 V8 虚拟机环境中编译和运行代码
  • Zlib- zlib，zlib模块提供Gzip和Deflate/inflate实现的压缩功能和解压处理

Node.js中的控制台

  • console.log([data][,…args])方法
    console.log方法用于标准输出流的输出

  • console.table(tabularData[,properties])方法
    console.log方法用于输出表格

  • console.time([label])和console.timeEnd([label])方法
    console.time启动一个定时器，用以计算一个持续时间，打印结果
    console.timeEnd停止之前通过调用console.time启动的定时器，并打印结果

  • console.trace([message][,…args])方法
    console.trace方法用于将当前位置处的栈信息作为标准信息进行输出

Node.js中的全局作用域及全局函数

全局作用域

Node.js中，全局作用域为global对象，代表Node.js中的全局命名空间，任何全局变量、函数或类都是该对象的一个属性值

全局函数
  • Buffer类，用于处理二进制数据
  • __dirname，非全局变量，作用域只在模块内，表示当前模块文件夹名称，等同于__filename的path.dirname()
  • __filenmae，非全局变量，作用域只在模块内，表示当前模块的文件名称—解析后的绝对路径
  • clearImmediate(immediateObject)，取消一个由setImmediate()创建的Immediate对象
  • clearInterval(intervalObject)，取消一个由setInterval() 创建的 Timeout 对象
  • clearTimeout(timeoutObject)，取消一个setTimeout()创建的Timeout对象
  • console类，用于打印
  • exports，非全局变量，作用域只在模块内，这是一个对于 module.exports 的更简短的引用形式
  • module，非全局变量，是每个模块本地的，module的自由变量是一个指向表示当前模块的对象的引用
  • process，提供当前Node.js进程的相关信息，以及控制当前 Node.js 进程
  • require()，非全局变量，引入模块
  • setImmediate(callback[,…args])，预定立即执行的 callback，它是在 I/O 事件的回调之后被触发。 返回一个用于 clearImmediate() 的 Immediate
  • setInterval(callback, delay[, …args])，预定每隔 delay 毫秒重复执行的 callback 返回一个用于 clearInterval() 的 Timeout
  • setTimeout(callback, delay[, …args])，预定在 delay 毫秒之后执行的单次 callback。 返回一个用于 clearTimeout() 的 Timeout
  • URL类，浏览器兼容的URL类，根据 WHATWG URL 标准实现。[解析URL的示例][]可以在标准本身里边找到
  • URLSearchParams类，URLSearchParamsAPI接口提供对URLquery部分的读写权限

Node.js的事件处理机制及事件环机制

大多数 Node.js 核心 API 都采用惯用的异步事件驱动架构，其中某些类型的对象（触发器）会周期性地触发命名事件来调用函数对象（监听器）。
例如，net.Server对象会在每次有新的连接时触发事件；fs.ReadStream会在文件被打开时触发事件；流对象 会在数据可读时触发事件。
所有能触发事件的对象都是EventEmitter类的实例。这些对象开放了一个eventEmitter.on()函数，允许将一个或多个函数绑定到会被对象触发的命名事件上。事件名称通常是驼峰式的字符串，但也可以使用任何有效的JavaScript属性名。
当EventEmitter对象触发一个事件时，所有绑定在该事件上的函数都被同步地调用。监听器的返回值会被丢弃。

EventEmitter类方法
  • emitter.addListener(eventName, listener)，对指定事件绑定事件处理函数，emitter.on(eventName, listener)的别名
  • emitter.on(eventName, listener)，添加listener函数到名为eventName的事件的监听器数组的末尾。不会检查listener是否已被添加。多次调用并传入相同的eventName和listener会导致listener被添加与调用多次
  • emitter.once(eventName, listener)，添加一个单次listener函数到名为eventName的事件。下次触发eventName时，监听器会被移除，然后调用
  • emitter.removeListener(eventName, listener)，从名为 eventName 的事件的监听器数组中移除指定的 listener。removeListener最多只会从监听器数组里移除一个监听器实例。如果任何单一的监听器被多次添加到指定eventName的监听器数组中，则必须多次调用removeListener才能移除每个实例。
  • emitter.removeAllListeners([eventName])，移除全部或指定eventName的监听器
  • emitter.emit(eventName[,…args])，按监听器的注册顺序，同步地调用每个注册名为eventName事件的监听器，并传入提供的参数
事件环机制

在Node.js中，采用非阻塞I/O机制，这意味着要求所有应用程序所进行的处理，如HTTP请求、数据库查询、文件的输入/输出等，都不会在处理结束之前阻碍其他处理的进行，也就是说，这些处理都是独立进行的，当处理结束时，会触发一个回调事件，也就是说，在Node.js中，我们所要编写的时各种I/O事件的回调函数的处理。在回调函数的执行过程中，他将转而处理新的事件，在该事件处理完毕之后，转而继续处理原回调函数。这种环状处理机制，在Node.js中称为事件环机制。
